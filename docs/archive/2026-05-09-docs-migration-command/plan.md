## Context
### Original Request
User wants a new command to migrate documentation from other workflow projects (OpenSpec, Spec Kit, Kiro, Cursor, Trae) and non-standard hand-maintained doc directories into OpenFlow's ADR-001 standard structure. The command should use a brainstorm-style interactive approach �?asking clarifying questions during migration, detecting document types and classifying them by semantics.

### Interview Summary
- **New command**, not an init upgrade; format: `/openflow-migrate-docs <source-docs-dir> [--target <target-dir>] [--dry-run]`
- **Brainstorm-style wizard**: report/clarify first, then execute; interactive `askQuestion` tool context
- **Copy-first disposal**: keep / move-to-references / delete; delete requires "DELETE ORIGINAL DOCS" phrase
- **Keep mode** �?Markdown + JSON report; **Delete mode** �?no report, no source metadata
- **Low-confidence** �?`references/raw` or `references/notes`, never auto-promote
- **High-confidence current facts** �?`docs/current/`; **ADR candidates** �?`docs/decisions/` (needs user confirmation)
- **Clarification** by topic/conflict batch, not per-file
- **Adapter priority**: OpenSpec > Spec Kit > Kiro > Cursor > Trae > generic
- **Content language** preserved; **paths** use English kebab-case
- **Read-only project context** for classification/naming/drift detection
- **Auto-init prompt** if source project has no OpenFlow structure
- **Confirmation gates**: apply, delete originals, overwrite existing, write decisions/, write conflicting current
- **Target defaults** to source project root, not current OpenFlow repo
- **State persistence**: `.sisyphus/docs-migration/{migration-id}.json`

### Metis Review (gaps addressed)
- **CI/Non-interactive**: Added `--dry-run` flag; non-interactive mode out of scope for v1
- **Confidence thresholds**: Defined as hardcoded: high �?.7, medium 0.4�?.7, low <0.4
- **Source path trust**: Must use `createSafePath`; reject symlinks and traversal; allow absolute/relative paths
- **Migration ID**: UUID-based; same source migrated twice �?resume prompt (new vs resume)
- **Decision confirmation**: Per-batch ADR confirmation; declined ADRs �?`references/notes/`
- **Mixed adapters**: Detect all, present to user for priority selection
- **Failure recovery**: Checkpoint per stage; resume from last checkpoint; no automatic rollback
- **Non-Markdown files**: Skip with warning in scan stage; do not convert
- **Concurrent runs**: Session lock via `active.json`; second run gets "already in progress" error
- **Language in filenames**: Always `.md` regardless of content language; no language tagging
- **Scope guards**: No auto-init execution, no content conversion, no ongoing sync, no template generation
