# OpenFlow - DevFlow OpenCode 插件

## SRS: Software Requirements Specification

---

## 1. 项目概述

### 1.1 项目名称

**OpenFlow** - 基于 oh-my-openagent 的开发工作流增强插件

### 1.2 项目类型

OpenCode 插件（独立项目，不修改 oh-my-openagent 源码）

### 1.3 核心目标

将 oh-my-openagent、Superpowers、OpenSpec 的最佳实践整合为一个独立的开发工作流插件，提供从需求到归档的完整流程支持。

### 1.4 目标用户

- 使用 oh-my-openagent 进行开发的团队
- 追求规范化开发流程的开发者
- 希望建立功能-代码映射知识库的团队

### 1.5 相关项目目录

- oh-my-openagent：F:\ai-code\oh-my-openagent
- superpowers：F:\ai-code\superpowers
- openspec：F:\ai-code\openspec
- opencode：F:\ai-code\opencode

## 3. 功能需求

### 3.1 阶段一：头脑风暴 (Brainstorming)

#### 3.1.1 功能描述

检测用户输入的新需求，建议运行 brainstorming skill。

#### 3.1.2 触发条件

- `brainstorming.enabled = true`
- `brainstorming.auto_trigger = true`
- 用户消息匹配需求模式
- 不存在现有的设计文档或执行计划

#### 3.1.3 需求检测模式

```typescript
const requirementPatterns = [
  /实现\s*(\S+)\s*功能/,
  /添加\s*(\S+)/,
  /创建\s*(\S+)/,
  /开发\s*(\S+)/,
  /implement\s+(\w+)\s+feature/i,
  /add\s+(\w+)/i,
  /create\s+(\w+)/i,
  /build\s+(\w+)/i,
]
```

#### 3.1.4 输出

- 控制台日志提示运行 `openflow-brainstorm` skill
- 不自动生成文档，由 skill 执行

#### 3.1.5 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `brainstorming.enabled` | boolean | true | 是否启用 |
| `brainstorming.output_dir` | string | docs/design | 设计文档目录 |
| `brainstorming.auto_trigger` | boolean | true | 自动检测新需求 |

---

### 3.2 阶段二：计划增强 (Plan Enhancement)

#### 3.2.1 功能描述

在 Prometheus 生成计划后，自动增强计划内容：添加 TDD 提示、添加验证任务。

#### 3.2.2 触发条件

- 检测到 `.sisyphus/plans/*.md` 文件写入
- 计划未被增强过

#### 3.2.3 TDD 扩展提示

在计划标题后插入 HTML 注释：

```html
<!-- OpenFlow TDD Expansion Hint
The following implementation tasks should follow TDD (Red-Green-Refactor):

- Task 1: 实现用户登录
- Task 2: 添加密码加密

For each implementation task:
1. RED: Write failing test first
2. GREEN: Implement minimal code to pass
3. REFACTOR: Clean up while keeping tests green
-->
```

#### 3.2.4 验证任务

在计划末尾添加验证阶段：

```markdown
---
## Verification Phase

### Security Checks
- [ ] **Secret Scan**: Check for accidentally committed secrets
- [ ] **Vulnerability Scan**: Run dependency vulnerability check

### Quality Checks
- [ ] **Lint Check**: Run linter
- [ ] **Type Check**: Run type checker
- [ ] **Test Suite**: Run all tests

> Auto-generated by OpenFlow. Complete all verification tasks before archiving.
```

#### 3.2.5 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `tdd.enabled` | boolean | true | 是否添加 TDD 提示 |
| `tdd.expand_threshold` | number | 3 | 实现任务数量阈值 |
| `verification.in_plan` | boolean | true | 是否添加验证任务 |
| `verification.security` | string[] | ["secret", "vuln"] | 安全检查项 |
| `verification.quality` | string[] | ["lint", "typecheck", "test"] | 质量检查项 |
| `verification.auto_fix` | boolean | false | 自动修复问题 |

---

### 3.3 阶段三：归档 (Archive)

#### 3.3.1 功能描述

用户手动触发归档，通过 `openflow-archive` skill 生成 SRS 文档。

#### 3.3.2 触发方式

```
skill(name="openflow-archive", feature="user-login")
```

#### 3.3.3 归档内容

```
docs/archive/{feature}/
├── srs.md                       # 软件需求规格（单一文件）
│
├── design/                      # 设计文档 (复制)
│   ├── proposal.md
│   ├── design.md
│   └── decisions.md
│
└── plan/                        # 执行计划 (复制)
    └── tasks.md
```

#### 3.3.4 SRS 文档结构

**srs.md** (由 AI 生成，包含代码映射):

```markdown
# {feature} - Software Requirements Specification

## 1. 概述
{从设计文档提取}

## 2. 业务流程
{从计划文档提取}

## 3. 代码实现

| 业务步骤 | 实现位置 | 关键代码 |
|----------|----------|----------|
| 用户登录 | src/auth/login.ts | handleLogin() (L12-45) |
| Token 生成 | src/auth/jwt.ts | generateToken() (L8-25) |

## 4. 修改的文件

| 文件 | 变更类型 |
|------|----------|
| src/auth/login.ts | created |
| src/auth/jwt.ts | created |

## 5. 相关文档
- 设计文档：docs/design/{feature}/
- 执行计划：.sisyphus/plans/{feature}.md
```

#### 3.3.5 数据来源

| 信息 | 来源 |
|------|------|
| 文件变更列表 | `client.session.messages()` → ToolPart |
| 代码符号 | `lsp_symbols` 工具（由 AI 调用） |
| 业务流程 | 计划文档（由 AI 解析） |
| 设计决策 | design/decisions.md（由 AI 解析） |

#### 3.3.6 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `archive.enabled` | boolean | true | 是否启用归档 |
| `archive.output_dir` | string | docs/archive | 归档目录 |

## 4. 安全设计

### 4.1 路径遍历防护

```typescript
// utils/security.ts
export function createSafePath(baseDir: string, ...segments: string[]): string {
  const resolvedBase = path.resolve(baseDir)
  const fullPath = path.resolve(resolvedBase, ...segments)

  if (!fullPath.startsWith(resolvedBase)) {
    throw new SecurityError(`Path traversal detected: ${segments.join('/')}`)
  }

  return fullPath
}
```

### 4.2 输入验证

| 输入 | 验证规则 |
|------|---------|
| Feature Name | 2-64字符, 仅 `a-z0-9-` |
| Config Path | 禁止 `..`, `\0`, 绝对路径, 系统目录 |
| Path Depth | 最大 10 层 |

## 5. Skills 系统

### 5.1 注册位置

Skills 动态注册到：
```
.opencode/skills/openflow/
├── openflow-brainstorm.md
├── openflow-verify.md
└── openflow-archive.md
```

### 5.2 可用 Skills

| Skill | 描述 |
|-------|------|
| `openflow-brainstorm` | 需求澄清和设计文档生成 |
| `openflow-verify` | 安全和质量检查 |
| `openflow-archive` | 功能归档和 SRS 生成 |

### 5.3 openflow-archive Skill 流程

```
1. 读取当前 session 的变更记录
   └── client.session.messages() → 提取 write/edit 工具调用

2. 读取设计文档和计划文档
   └── docs/design/{feature}/
   └── .sisyphus/plans/{feature}.md

3. 调用 LSP 获取代码符号（可选）
   └── lsp_symbols 工具

4. AI 生成 SRS 文档
   └── 整合以上信息
   └── 建立业务-代码映射
   └── 写入 docs/archive/{feature}/srs.md

5. 复制相关文档
   └── design/ → docs/archive/{feature}/design/
   └── plan/ → docs/archive/{feature}/plan/
```

## 11. 用户流程

### 11.1 完整流程

```
1. 用户: "实现用户登录功能"
   │
   ▼
2. [自动] Brainstorming Hook 检测
   ├── 匹配需求模式
   ├── 检查是否存在设计/计划
   └── 建议运行 openflow-brainstorm skill
   │
   ▼
3. 用户: skill(name="openflow-brainstorm")
   ├── 澄清需求
   ├── 生成 2-3 种方案
   └── 生成 docs/design/user-login/
   │
   ▼
4. [自动] Prometheus 生成计划
   │
   ▼
5. [自动] Plan Enhancement Hook
   ├── 添加 TDD 提示
   └── 添加验证任务
   │
   ▼
6. 用户: /start-work user-login
   │
   ▼
7. Sisyphus 执行
   ├── Task 1: RED → 写测试
   ├── Task 2: GREEN → 写代码
   ├── Task 3: REFACTOR
   └── Task N: 验证任务
   │
   │  (OpenCode 自动记录所有变更到 session)
   │
   ▼
8. 用户: skill(name="openflow-archive", feature="user-login")
   │
   ▼
9. AI 执行归档
   ├── 从 session 读取变更记录
   ├── 读取设计文档和计划
   ├── 调用 LSP 获取代码符号
   ├── 生成 SRS 文档
   └── 复制相关文档到归档目录
```
