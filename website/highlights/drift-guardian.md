# 漂移检测与 SDD

本文档介绍 OpenFlow 的检查点式漂移检测（Drift Guardian）和语义漂移检测（SDD）机制。

## 核心理念

OpenFlow 不采用常驻后台监控的方式检测漂移，而是在自然的检查点（plan milestone）进行语义级别的漂移审查。

## 检查点式漂移检测

### 工作流程

```
功能设计 → design.md/behavior.md 生成合约标记（Contract Markers）
         → /openflow-writing-plan 读取标记，嵌入检查点任务到实施计划
         → 编码阶段：仅轻量文件变更记录，无 LLM 调用
         → 里程碑完成：异步语义漂移审查（对比代码与合约标记）
         → 建议性输出（不阻塞 AI 执行）
```

### 关键特性

- **合约标记**：在设计阶段生成，明确代码必须满足的断言
- **计划嵌入**：检查点任务作为实施计划的一部分
- **异步审查**：里程碑完成后运行，不阻塞编码
- **建议性**：语义检查产生建议性证据，不替代 `quality-gate` 的最终判断

## 闭环追溯

归档时，`implementation-mapper` 使用标记 ID 构建完整追溯链：

```
标记 → 计划任务 → 检查点证据 → 验证结果 → 代码
```
